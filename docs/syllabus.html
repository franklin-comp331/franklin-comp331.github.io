Syllabus: COMP 331: AI-Assisted Software Engineering

This syllabus is referenced in the Course Orientation and provides the full course structure, learning outcomes, topics, grading, and weekly schedule. For tool installation and setup, see the Tool Setup Guide.

Course Name/Number: COMP 331: AI-Assisted Software Engineering
Format: 12-week
Credits: 4
Prerequisite: COMP 311 – Advanced Java Programming or equivalent programming experience

Course Description

This course explores how software engineers can use AI-assisted tools intentionally while building controlled, evaluable coding-agent workflows. Students will use GitHub Copilot, Java, GitHub, testing tools, evaluation criteria, and structured documentation to move from basic AI-assisted coding toward a complete Controlled Coding Agent final project. Emphasis is placed on structured task input, eval-driven development, human oversight, risk analysis, reproducibility, and critically evaluating AI-generated outputs.

Learning Outcomes

By the end of the course, students will be able to: - Set up and verify a Java development environment with GitHub Copilot and GitHub-based submission workflows. (Apply) - Document AI-assisted development decisions through weekly interaction logs. (Analyze) - Translate feature requests into structured task specifications for AI-assisted implementation. (Create) - Define and run evaluations that measure correctness and detect regressions in AI-generated outputs. (Evaluate) - Integrate structured input, AI-assisted generation, evaluation, and human approval into a controlled coding-agent workflow. (Create) - Analyze reliability, security, maintainability, and production-readiness risks in AI-assisted systems. (Analyze / Evaluate) - Deliver a complete Controlled Coding Agent final project with documentation, evaluation evidence, and a reproducible demo. (Create)

Topics Covered

  • Environment setup and AI-assisted software engineering foundations
  • GitHub Copilot, GitHub Education, Git, Java IDEs, and repository-based submission workflows
  • Weekly AI interaction logging and evidence-based reflection
  • Structured AI workflows and task specifications
  • Eval-driven development, measurable correctness criteria, and regression detection
  • Integrated AI system implementation with structured input, generation, and evaluation
  • Risk analysis, reliability, security, maintainability, and production readiness
  • Controlled coding-agent workflows with human oversight checkpoints
  • Final Project proposal, architecture, partial implementation, evaluation strategy, complete system, and demo

Textbooks & Materials

  • Selected academic journal and conference papers on AI in software engineering
  • Coursera specialization: Generative AI for Software Engineering
  • Official GitHub Copilot documentation and tutorials
  • Practical guides on prompt engineering and AI tooling focused on Copilot
  • Supplementary readings on evaluation, software testing, reproducibility, responsible AI, and software security

Tools and Software

See the Tool Setup Guide for installation and configuration walkthroughs.

  • GitHub Copilot (primary AI tool integrated into Java IDEs). Students should set up GitHub Education with their university email address at github.com/settings/education/benefits to access education benefits, including Copilot access when approved.
  • Visual Studio Code or IntelliJ IDEA with GitHub Copilot plugin installed
  • Latest JDK
  • Git and GitHub for version control and project collaboration
  • JUnit 5 for Java unit testing
  • Maven or Gradle for build and dependency management

Assignment Schedule & Assessments

Assignment Weeks Title Points Description
Assignment 1 1 Environment Setup & AI-Assisted SWE Foundations 30 Set up the development environment, verify Copilot, create a minimal Java project, and reflect on Week 1 concepts
Weekly Log 2–10 Weekly AI Interaction Log 90 Maintain weekly evidence of prompting, iteration, evaluation, and human engineering decisions
Lab 1 3 Structured AI Workflow 90 Build the structured task input layer and compare naive vs structured AI-assisted development
Lab 2 5 Eval-Driven Development 100 Add measurable correctness criteria and regression checks for AI-generated outputs
Lab 3 7 AI System Implementation 100 Integrate structured input, AI generation, and evaluation into a cohesive workflow
Lab 4 9 AI Risk Analysis and Production Readiness 90 Analyze failure modes, reliability, security, maintainability, and production readiness
Lab 5 10 Coding Agents and Controlled Workflows 100 Implement a controlled multi-step agent loop with human oversight
Final Milestone 1 10 Proposal & Architecture 80 Define scope, architecture, structured workflow, and development plan for the final project
Final Milestone 2 11 Partial Implementation, Evaluation Strategy, and Identified Risks 140 Demonstrate meaningful implementation progress, evaluation strategy, and risk analysis
Final Submission 12 Complete System, Documentation, Evaluation Report, and Demo 180 Deliver the complete Controlled Coding Agent system with documentation, evaluation, and demo

Total Points: 1000

Unless otherwise specified, assignments are due at the end of the listed week. The Weekly AI Interaction Log is submitted once per week from Week 2 through Week 10.

Grading Breakdown

Component Points
Assignment 1: Environment Setup & AI-Assisted SWE Foundations 30
Weekly AI Interaction Log 90
Labs 1-5: Controlled Coding Agent workflow development 480
Final Project Milestones and Final Submission 400
Total 1000

Assignments are weighted to build progressively from setup and weekly reflection into labs, controlled coding-agent workflows, and a complete final project.

Submission Expectations

  • For repository-based assignments, submit the repository link and Git commit hash for the exact version submitted. Also submit Canvas links, screenshots, reports, or demo links as requested in each assignment.
  • Maintain AI usage evidence, including prompts, Copilot interactions, accepted or rejected outputs, evaluations, and human decisions.
  • Keep README files current with setup, run, evaluation, and limitations information.
  • The Weekly AI Interaction Log is submitted weekly from Week 2 through Week 10.
  • Final Project submissions must be reproducible from a clean environment and include evaluation evidence.
