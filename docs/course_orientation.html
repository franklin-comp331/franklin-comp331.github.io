1) Welcome & Course Introduction

Instructor: Add name, office hours, contact preferences

Format: 12-week · 4 credits · Prerequisite: COMP 311 | Advanced Java Programming (or equivalent)

Why this course now?

AI-assisted development is changing how software is built. You’ll learn to pair your expertise with tools like GitHub Copilot to generate code, tests, docs, and plans—while critically evaluating quality, security, and ethics.

Course arc (high-level):

  • Week 1: Environment setup and AI-assisted software engineering foundations
  • Weeks 2–10: Weekly AI Interaction Log documenting prompts, decisions, and evaluations
  • Weeks 3, 5, 7, 9, and 10: Labs that build a Controlled Coding Agent workflow
  • Weeks 10–12: Final Project proposal, partial implementation, evaluation strategy, risk analysis, final system, and demo

2) Syllabus & Learning Outcomes

See the Syllabus & Weekly Schedule for full course description, learning outcomes, topics, grading breakdown, and weekly activities.

Catalog Description

Explores how software engineers can use AI-assisted tools intentionally while building controlled, evaluable coding-agent workflows. Emphasis is placed on structured task input, AI-assisted code generation, evaluation, risk analysis, human oversight, reproducibility, and professional documentation. Hands-on labs build toward a final Controlled Coding Agent project in Java.

Learning Outcomes (mapped to Bloom’s levels):

  1. Set up and verify a Java development environment with GitHub Copilot and GitHub-based submission workflows. (Apply)
  2. Document AI-assisted development decisions through weekly interaction logs. (Analyze)
  3. Translate feature requests into structured task specifications for AI-assisted implementation. (Create)
  4. Define and run evaluations that measure correctness and detect regressions in AI-generated outputs. (Evaluate)
  5. Integrate structured input, AI-assisted generation, evaluation, and human approval into a controlled coding-agent workflow. (Create)
  6. Analyze reliability, security, maintainability, and production-readiness risks in AI-assisted systems. (Analyze/Evaluate)
  7. Deliver a complete Controlled Coding Agent final project with documentation, evaluation evidence, and a reproducible demo. (Create)

5) AI Use, Evidence, and Academic Integrity

GitHub Copilot is the approved AI assistant for course programming work unless an assignment says otherwise. You may use Copilot to help draft code, tests, documentation, evaluation criteria, and workflow artifacts, but you remain responsible for every submitted artifact.

Required Evidence

  • Record meaningful Copilot interactions in the Weekly AI Interaction Log.
  • Identify the Copilot mode used, such as inline suggestion, chat, edit, agent/workspace, test generation, review, or debugging.
  • Describe AI assistance in each repository README or assignment documentation.
  • Preserve prompts, accepted or rejected suggestions, manual edits, evaluations, verification methods, and final decisions when relevant.

Not Allowed

  • Submitting AI-generated work without acknowledgment.
  • Submitting code you do not understand or cannot explain.
  • Sharing full solutions with classmates unless the professor explicitly makes an activity collaborative.
  • Pasting secrets, private data, proprietary code, or protected information into AI tools.

6) Risks To Watch Throughout the Course

The labs and Final Project ask you to evaluate AI-assisted systems, not just produce code. Keep these risks visible in your weekly logs, evaluations, and project documentation.

  • Correctness: Generated code may appear plausible while missing edge cases or requirements.
  • Security: AI suggestions may introduce unsafe input handling, secret leakage, weak cryptography, or injection risks.
  • Maintainability: Generated code may be hard to read, overcomplicated, or poorly aligned with your architecture.
  • Evaluation gaps: Passing one demo does not prove the workflow is reliable. Define measurable checks.
  • Human oversight: Controlled workflows need clear points where a human reviews, approves, rejects, or revises AI output.

7) Learning Activities

Controlled Workflow Pattern

  1. Specify: Translate a request into structured input with goals, constraints, edge cases, and acceptance criteria

  2. Generate or revise: Use Copilot inline suggestions, Chat, edit/review features, or agent-style assistance to propose implementation, tests, documentation, or workflow artifacts

  3. Evaluate: Run tests, compiler checks, scripts, manual reviews, or structured comparisons against explicit criteria

  4. Control: Add human approval checkpoints before accepting important changes

  5. Document: Preserve prompts, outputs, evaluation results, risks, and final decisions

This pattern appears repeatedly in Labs 1-5 and the Final Project. The goal is not to maximize AI usage; the goal is to make AI-assisted development controlled, observable, evaluable, and reproducible.

8) Success Strategies

  • Keep the Weekly AI Interaction Log current: record Copilot mode, prompts, outputs, verification methods, decisions, and what worked

  • Prefer small, precise prompts over broad requests; iterate quickly

  • Define evaluation criteria before trusting generated code; use AI to scaffold edge cases where appropriate

  • Use tests, static analysis, security checks, and manual review alongside AI suggestions

  • Commit early/often; preserve artifacts that make workflow decisions reproducible

  • Treat AI like a junior pair-programmer: useful for drafts, but never the final authority
