Lab 1: Structured AI Workflow

Course Name/Number: COMP 331: AI-Assisted Software Engineering
Week: 3
Points: 90

Complete this lab using a structured AI-assisted workflow.

1 Purpose of Assignment

This assignment is intended to help you:

  • Establish the task input layer of your coding agent
  • Translate feature requests into structured specifications
  • Compare naive vs structured AI-assisted development
  • Begin building the foundation of a Java-based coding agent system

2 Steps / Requirements of Assignment

Build the first layer of a Controlled Coding Agent workflow: a Java command-line program that can accept a user request, send it to an LLM, and compare naive prompting with structured prompting. This lab should feel small and controlled. You are not building the full agent yet.

Start fresh for this lab. Create a new repository named comp331-java-agent. Do not use your Assignment 1 setup repository as the starting point unless your professor explicitly approves it. Create this lab repository once; it is the continuing repository for Labs 1-5 and may become the starting point for your final project.

  1. Create a fresh GitHub repository named comp331-java-agent. You will continue developing in this same repository for Labs 2-5; do not create a new comp331-java-agent repository for each lab.
  2. Create a Java project using Maven in that repository. A simple structure such as src/main/java/.../Main.java is enough.
  3. Configure your Gemini API key using the Gemini API Setup Guide. Do not commit API keys, .env files, or other secrets.
  4. Write a Main class that first sends one hard-coded prompt to the model and prints the response. This verifies that your Java project can connect to the API.
  5. Revise the program so it accepts the user prompt from command-line arguments, for example mvn exec:java -Dexec.args="Explain this Java method".
  6. Add a simple message representation for later labs. This may be a small Message class or record with fields such as role and content.
  7. Add a --verbose flag or similar option that prints helpful debugging information, such as the user prompt, selected model, and token counts if your API provides them.
  8. Choose a small Java feature request that can be implemented in a utility class, such as formatting a string, validating input, parsing a simple expression, or computing a value.
  9. Ask GitHub Copilot or another approved coding assistant for help once using a naive prompt or informal comment. Save the prompt and resulting code or describe the interaction.
  10. Create a structured task specification with goal, inputs, outputs, constraints, edge cases, and acceptance criteria. Use that specification to revise or re-implement the feature.
  11. Compare the naive and structured approaches in your README. Focus on concrete differences in output quality, missing edge cases, and human decisions.

Suggested checkpoint commands:

mvn test
mvn exec:java -Dexec.args="Why are structured prompts useful for Java development?"
mvn exec:java -Dexec.args="--verbose Explain what this program does"

4 Using GitHub Copilot

Use GitHub Copilot to compare unstructured assistance with intentional, structured assistance. You remain responsible for understanding, testing, and explaining all submitted work.

Required Copilot workflow:

  • Use inline Copilot once with a vague comment or naive prompt. Save the prompt/comment and result.
  • Use Copilot Chat again with a structured task specification. Include goal, inputs, outputs, constraints, edge cases, and acceptance criteria.
  • Ask Copilot to explain the structured result before accepting it.
  • Compare the naive and structured outputs in your README. Focus on correctness, missing edge cases, and what you changed manually.

5 Deliverables

Your repository should include:

  • The Lab 1 repository named comp331-java-agent, which will continue as your working repository for Labs 2-5.
  • A runnable Java Maven project with a command-line entry point.
  • Code that sends a prompt to an LLM from Java.
  • Command-line prompt handling and a verbose/debug mode.
  • Java source code for both the naive and structured feature attempts, or clearly separated commits showing each approach.
  • A structured task specification.
  • A README.md explaining setup, API key configuration, run commands, and the naive vs structured comparison.
  • Copilot evidence showing the naive interaction, structured interaction, accepted changes, rejected or revised suggestions, and verification method.
  • Evidence of AI-assisted development through prompts, comments, screenshots, transcript snippets, or log links.
