Lab 4: AI Risk Analysis and Production Readiness

Course Name/Number: COMP 331: AI-Assisted Software Engineering
Week: 9
Points: 90

This lab analyzes the system built in Lab 3: AI System Implementation. Continue maintaining the Weekly AI Interaction Log.

1 Purpose of Assignment

This assignment is intended to help you:

  • Analyze failure modes in your coding agent
  • Evaluate reliability, security, and maintainability
  • Propose improvements to increase robustness
  • Strengthen the safety controls around Java tools before adding an autonomous loop

2 Steps / Requirements of Assignment

Conduct a risk analysis of your current Java coding agent workflow. Focus on realistic failures in your own implementation rather than generic AI risks. Your system now has tools that can inspect files, read files, write files, and run Java evaluation commands, so your analysis should be specific to those capabilities.

  1. Review each tool from Lab 3 and identify what could go wrong if an LLM requested unsafe or incorrect arguments.
  2. Identify at least six failure modes. Include path traversal, accidental overwrite, command execution risk, secret exposure, misleading model output, incomplete tests, and human oversight gaps where relevant.
  3. Analyze reliability, security, maintainability, and human oversight risks. At least one risk must come from each category.
  4. Rate each risk by likelihood and impact using a simple scale such as Low, Medium, or High.
  5. Propose concrete mitigations. Examples include path normalization, read-size limits, command allowlists, write approval checkpoints, max iteration limits, logging, and better test coverage.
  6. Implement at least two small safeguards or tests based on your analysis. Reasonable options include blocking absolute paths, adding a command allowlist, adding a write confirmation mode, or adding tests for secret-like files.
  7. Add a production-readiness checklist that distinguishes what is currently safe enough for coursework from what would be required before real use.
  8. Identify which mitigations you plan to carry into Lab 5 and the final project.

Suggested risk table columns:

Risk Tool/Component Category Likelihood Impact Current Control Proposed Mitigation

Suggested checkpoint commands:

mvn test
mvn test -Dtest=*Tool*

3 Action Items

  • Click the link to the full hosted lab instructions: Lab 4: AI Risk Analysis and Production Readiness.
  • Follow the instructions in the lab page.
  • Open your existing comp331-java-agent repository and review each Lab 3 tool implementation.
  • Complete the risk analysis, safeguard implementation, and safety-test work described in the linked instructions.
  • Document the risk table, production-readiness checklist, prioritized improvement plan, Copilot-assisted analysis notes, and verification results in the repository.
  • Commit your completed work and obtain the Git commit hash for the submitted version.
  • Submit the repository link, commit hash, and risk-analysis document through Canvas.

4 Using GitHub Copilot

Use GitHub Copilot as a risk-analysis assistant, not as the final authority. Your risk analysis must be specific to your own Java tool implementation.

Recommended Copilot workflow:

  • Ask Copilot to identify failure modes for your actual file, path, write, and test-execution tools.
  • Verify Copilot’s suggestions against your code and tests. Remove generic risks that do not apply.
  • Ask Copilot to critique your proposed mitigations, then decide which safeguards are realistic for this project.
  • Include one example where Copilot missed, understated, or overstated a risk.

Suggested prompt:

Review this Java coding-agent tool design for realistic risks.
Focus on reliability, security, maintainability, and human oversight.
Identify concrete failure modes tied to file reading, file writing, path handling, and test execution.
Do not give generic AI risk language.

5 Deliverables

Your repository should include:

  • Risk analysis document or README section specific to your Java coding agent.
  • Failure mode table with likelihood, impact, current controls, and mitigation.
  • Evidence of at least two implemented safeguards or safety tests.
  • Production-readiness checklist.
  • Prioritized improvement plan for Lab 5 and the final project.
  • Copilot risk-analysis notes, including one example where you corrected or refined Copilot’s analysis.

6 Grading

Production-readiness plan is actionable 5
Copilot-assisted risk analysis is verified and corrected by the student 5
