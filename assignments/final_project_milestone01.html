Final Project Milestone 1: Proposal & Architecture

Course Name/Number: COMP 331: AI-Assisted Software Engineering
Week: 10
Points: 80

This milestone builds on Labs 1-5, especially Lab 5: Coding Agents and Controlled Workflows.

1 Purpose of Assignment

This assignment is intended to help you:

  • Define the scope of your Controlled Coding Agent
  • Design a clear system architecture based on Labs 1-5
  • Establish a structured AI-assisted workflow
  • Create a realistic development plan for completion

2 Steps / Requirements of Assignment

Prepare a proposal and architecture plan for your final Controlled Coding Agent system. You may continue your lab repository or create a dedicated final project repository.

  1. Define the problem your coding agent addresses.
  2. Describe intended users and expected workflow.
  3. Create an architecture diagram or structured architecture description.
  4. Identify core components, including input, generation, evaluation, workflow, and persistence.
  5. Create a development plan for Weeks 10-12.

3 Using GitHub Copilot

Use GitHub Copilot as a planning and critique partner, not as a substitute for your architecture decisions. You remain responsible for scope, feasibility, and tradeoffs.

Recommended Copilot workflow:

  • Ask Copilot Chat to critique your project scope for feasibility in the remaining course timeline.
  • Ask Copilot to generate two or three architecture alternatives, then choose and justify one.
  • Ask Copilot for likely risks, evaluation strategy ideas, and a milestone breakdown.
  • Document at least one Copilot-generated alternative that you did not choose and explain why.

4 Action Items

  • Click the link to the full hosted milestone instructions: Final Project Milestone 1.
  • Follow the instructions in the milestone page.
  • Go to github.com and either continue your lab repository or create a dedicated final project repository.
  • Create the proposal, architecture documentation or diagram, development plan, and milestone checklist described in the linked instructions.
  • Include AI-assisted planning notes showing Copilot prompts, alternatives considered, accepted ideas, rejected ideas, and your final decision.
  • Submit the repository link and paths to the proposal and architecture documentation through Canvas.

5 Deliverables

Your repository should include:

  • Project proposal.
  • Architecture documentation or diagram.
  • Development plan and milestone checklist.
  • AI-assisted planning notes showing at least one Copilot-generated alternative and your decision.
  • Initial implementation artifacts if available.

6 Grading

Category Points
Project scope and problem statement are clear 20
Architecture covers major system components 25
Structured AI-assisted workflow is defined 15
Development plan is realistic and specific 10
Copilot-assisted planning evidence shows alternatives and student judgment 5
Repository organization is ready for continued work 5
Total 80

7 Submission Instructions

Submit your project repository link through Canvas. Include links or paths to the proposal, architecture documentation, and Copilot-assisted planning notes.
